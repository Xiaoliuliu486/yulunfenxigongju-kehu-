# 爱敬舆论分析 - Coze 工作流部署指南

## 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                     Coze 工作流 (Agent)                       │
│                                                             │
│  用户上传Excel                                               │
│       ↓                                                      │
│  ┌──────────────────────────────────────────────────┐        │
│  │ 插件节点: 爱敬舆论分析 API                          │        │
│  │   → POST /analyze-sync 上传Excel                   │        │
│  │   → 返回完整分析结果（含负面评价列表、统计报告）      │        │
│  └──────────────────────────────────────────────────┘        │
│       ↓                                                      │
│  ┌──────────────────────────────────────────────────┐        │
│  │ LLM节点: 解读分析结果                              │        │
│  │   → 把API返回的数据，转成客户友好文案               │        │
│  │   → 突出关键发现、给出建议                         │        │
│  └──────────────────────────────────────────────────┘        │
│       ↓                                                      │
│  输出给客户（图文并茂的报告）                                  │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                  Railway 云服务器 (免费)                      │
│                                                             │
│  FastAPI 服务                                                │
│  ├── POST /analyze-sync  ← Coze插件调用此接口               │
│  ├── POST /analyze       ← 异步（备用）                     │
│  ├── GET  /status/{id}   ← 查询进度                         │
│  ├── GET  /result/{id}   ← 获取结果                         │
│  └── GET  /health        ← 健康检查                         │
│                                                             │
│  功能: 读取Excel → 爬取链接 → 情感分析 → 生成报告            │
└─────────────────────────────────────────────────────────────┘
```

## 第一步：部署 API 到 Railway（免费）

### 前置准备

1. 注册 [Railway](https://railway.app/)（可以用 GitHub 账号登录）
2. 安装 [Git](https://git-scm.com/)

### 部署步骤

#### 方式 A：通过 Railway CLI 部署（推荐，最快）

```bash
# 1. 安装 Railway CLI
npm install -g @railway/cli

# 2. 登录 Railway
railway login

# 3. 进入项目目录
cd d:\agent\fenxi_agent

# 4. 初始化 Railway 项目
railway init

# 5. 部署
railway up

# 6. 生成域名
railway domain
```

部署成功后，Railway 会分配一个域名，例如：`https://aijing-analysis.up.railway.app`

#### 方式 B：通过 GitHub + Railway Dashboard 部署

1. **在 GitHub 上创建仓库**
   ```bash
   cd d:\agent\fenxi_agent
   git init
   git add .
   git commit -m "initial commit"
   ```
   然后在 GitHub 上创建仓库并推送代码

2. **在 Railway Dashboard 中部署**
   - 打开 https://railway.app/dashboard
   - 点击 "New Project" → "Deploy from GitHub repo"
   - 选择你刚推送的仓库
   - Railway 会自动检测 Python 项目，安装依赖（`pip install -r requirements.txt`）
   - 自动运行启动命令（`uvicorn api_server:app --host 0.0.0.0 --port $PORT`）

3. **生成公网域名**
   - 在 Railway Dashboard 中，点击项目
   - 进入 "Settings" → "Domains"
   - 点击 "Generate Domain"
   - 获得域名，例如：`https://aijing-analysis.up.railway.app`

### 验证部署

用浏览器访问 `https://你的域名/health`，应该返回：

```json
{"status":"healthy","active_tasks":0,"timestamp":"2026-04-28T..."}
```

## 第二步：在 Coze 中创建插件

1. **打开 Coze 工作台** → [https://www.coze.cn](https://www.coze.cn)
2. **进入"插件"页面** → 点击"创建插件"
3. **填写插件信息**：
   - 插件名称：`爱敬舆论分析`
   - 插件描述：`上传Excel自动分析多平台负面舆论评价`
   - 插件图标：可以上传一个自定义图标
4. **选择接入方式** → "OpenAPI"
5. **上传 OpenAPI 文件**：
   - 首先修改 `coze_plugin/openapi.yaml` 中的 `servers.url`
   - 将 `https://your-server.com` 替换为你的 Railway 域名
   - 例如：`https://aijing-analysis.up.railway.app`
   - 上传修改后的 `openapi.yaml` 文件
6. **鉴权方式** → 选择"无鉴权"（No Auth）
7. **点击"保存"** → 插件创建完成

## 第三步：在 Coze 中创建 Bot

### 3.1 创建 Bot

1. **进入 Bot 页面** → 点击"创建 Bot"
2. **填写 Bot 信息**：
   - Bot 名称：`爱敬舆论分析助手`
   - Bot 描述：`上传Excel文件，自动分析多平台评价中的负面舆论`
   - 头像：可自定义

### 3.2 配置人设与回复逻辑

```
# 角色
你是爱敬品牌的专业舆论分析助手。你的职责是帮助用户分析多平台（抖音、快手、小红书、今日头条、懂车帝等）的消费者评价，识别负面舆论，并生成清晰的分析报告。

# 工作流程
1. 用户上传Excel文件后，调用"爱敬舆论分析"插件进行分析
2. 接收插件返回的分析结果
3. 将结果整理成清晰、易读的报告呈现给用户

# 报告格式要求
当你拿到分析结果后，请按以下格式整理报告：

## 📊 总体概况
- 总分析条数：{total} 条
- 重复数据：{duplicates} 条
- 负面评价：{negative_count} 条（{negative_rate}）
- 处理耗时：{processing_time}

## 📋 置信度分布
- 高置信度：{high} 条（负面特征明显）
- 中置信度：{medium} 条（有一定负面特征）
- 低置信度：{low} 条（需人工复核）

## 📱 各平台负面分布
{按平台列出数量}

## 🔑 高频负面关键词TOP10
{列出关键词及出现次数}

## ⚠️ 重点负面评价
{逐条列出，包含摘要、平台、置信度}

## 💡 建议
基于分析结果给出1-3条建议

# 注意事项
- 如果分析结果中没有负面评价，应回复"本次分析未发现明显的负面舆论评价"
- 不要对分析结果进行过度解读
- 保持客观中立的态度
- 如果调用插件出错，请友好的告诉用户错误原因并建议重试
```

### 3.3 添加技能（插件）

1. 在 Bot 编辑页面，找到"技能"区域
2. 点击"添加技能" → "插件"
3. 在插件列表中找到你刚创建的"爱敬舆论分析"插件
4. 点击添加

### 3.4 保存并发布

1. 点击"保存"按钮
2. 点击"发布" → 可以选择发布到"扣子商店"或"仅自己使用"
3. 如果是给客户使用，建议发布后生成分享链接

## 第四步：测试 Bot

1. 打开 Bot 对话界面
2. 上传一个 Excel 文件（示例格式见下方）
3. 等待 Bot 调用插件并返回结果
4. 检查分析报告是否完整、准确

### Excel 文件格式要求

| 摘要 | 链接 | 平台 |
|------|------|------|
| 这个产品很好用，回购多次 | https://www.douyin.com/... | 抖音 |
| 质量太差了，用一次就坏了 | https://www.xiaohongshu.com/... | 小红书 |
| 性价比不错，推荐购买 | https://www.kuaishou.com/... | 快手 |

**列名自动识别**（支持任意一种）：
- 摘要列：`摘要`、`评价内容`、`标题`、`评论`、`内容`
- 链接列：`链接`、`url`、`网址`、`link`
- 平台列：`平台`、`来源`、`渠道`

## 第五步：故障排查

### API 调用失败

1. **检查 Railway 服务状态**
   ```
   访问 https://你的域名/health
   ```
   如果返回 200，说明服务正常
   如果无法访问，检查 Railway Dashboard 中的日志

2. **检查域名配置**
   ```
   确保 openapi.yaml 中的 servers.url 已正确替换为你的 Railway 域名
   ```

3. **查看 Railway 日志**
   - 在 Railway Dashboard 中点击项目
   - 进入 "Deployments" → 点击最新部署
   - 查看 "Logs" 标签

### 分析结果不准确

- 检查 Excel 文件的列名是否被正确识别
- 检查摘要列是否包含足够的文本内容
- 如果负面评价过多或过少，可以在 Bot 人设中调整判断标准

## 方案对比总结

| 特性 | Coze 工作流方案 | Python GUI/EXE 方案 |
|------|----------------|-------------------|
| 使用方式 | 网页端对话上传 | 双击 exe 运行 |
| 部署要求 | 需要部署 API 到 Railway | 无需部署，直接运行 |
| 适合用户 | 习惯用网页/聊天的客户 | 习惯用软件的客户 |
| 数据安全 | 数据经过第三方服务器 | 本地处理，数据不外传 |
| URL 爬取 | 依赖 API 服务（部署版） | 本地爬取 ✅ |
| 分析能力 | 可用豆包大模型增强 | 规则引擎（关键词+SnowNLP） |
| 交互体验 | 对话式，更灵活 | 界面稳定，功能固定 |

两个方案可以同时提供，让客户自由选择。
